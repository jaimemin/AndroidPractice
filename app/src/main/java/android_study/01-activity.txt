Activity
- 화면을 구성하는 가장 기본이 되는 요소
- 자동으로 activity와 매칭되는 xml을 생성 + manifest 등록
- 화면을 그리는 기능 + 화면에서 발생할 수 있는 일들을 처리할 수 있는 기능 내장

- 특수한 Activity
    - 런처 activity, main activity
        - 앱이 실행될 때 최초로 실행되는 activity

- 생명 주기 (Life Cycle): 안드로이드 OS가 activity를 직접 종료시키는 일은 없다 (프로세스 종료는 가능)
    - onCreate
        - 필수적으로 구현을 해야함
        - 생명주기 중에서 단 한번만 발생
            - 단 한번만 하면 되는 일들을 여기에서 구현
            - ex) 화면 그리기, 데이터 준비
    - onStart
        - activity를 foreground로 보내 상호작용할 수 있도록 준비
            - foreground: 사용자 눈에 보일 때 또는 사용 중일 때
        - 매우 빠르게 완료
    - onResume
        - 앱이 사용자와 상호작용이 가능한 상태
        - 특정 이벤트가 발생하여 앱에서 포커스가 떠날 때까지 이 상태에 머무르게 된다
            - ex) 전화가 온다거나 기기 화면이 꺼질 때
            - 화면이 가려지지 않으면 이 상태에 머뭄
    - onPause
        - 사용자가 activity를 떠나는 것을 나타내는 첫 번째 신호
            - 액티비티가 foreground에 있지 않다는 신호
        - 잠시 작업을 중지해야하는 작업을 일시 중지
            - ex) 영상 일시 중지, GPS 일시 중지
    - onStop
        - activity가 더 이상 사용자에게 표시가 되지 않는 상태
            - 새로운 activity가 나오는 경우
            - 화면이 가려지는 경우
        - 사용자에 보이지 않는 동안 앱은 리소스를 해제 해야함
            - ex) 영상 종료, GPS 종료
        - 마지막으로 해야 할 일 등을 수행
            - ex) 사용자가 수행한 작업을 저장
    - onDestroy
        - activity가 소멸되기 전에 호출
            - 기기 회전 or 멀티 윈도우 시스템이 일시적으로 activity를 소멸시킴
            - onStop에서 해제되지 않은 모든 리소스 해제
